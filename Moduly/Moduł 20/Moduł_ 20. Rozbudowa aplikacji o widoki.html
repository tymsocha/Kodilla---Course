20.1. HTML - Co to jest?

W tym module zmierzymy się z podstawowymi technologiami, które towarzyszą osobom tworzącym strony internetowe - HTML oraz CSS.

Powiedzmy sobie na samym początku co to jest HTML. Nie ma co ukrywać, że jest to najbardziej kojarzona przez wszystkich technologia związana ze światem IT, wielu miało podstawy HTML w szkole podstawowej i średniej. Pełna nazwa skrótu HTML to HyperText Markup Language, czyli w skrócie - język znaczników (tagów) oraz hiperłącz.

Co kryje się pod określeniem "język znaczników"? Podobnie jak w XML, cała zawartość pliku HTML to nic innego niż konkretny zbiór tagów, które połączone ze sobą dają szablon, który jest rozumiany / interpretowany przez przeglądarki internetowe czy też inne programy. Pozwala to na stworzenie wizualnej strony aplikacji - w takim kontekście będziemy patrzeć na HTML.

Hiperłącza natomiast, są to po prostu linki do innych fragmentów strony internetowej bądź do innych stron internetowych.

Przejdźmy do konkretów i omówmy poniższy kod:

<!DOCTYPE html>
<html>
    <head>
        <title>Kodilla</title>
    </head>
    <body>
        <h1>Kodilla Course</h1>
        <p>Something new!</p>
    </body>
</html>

<!DOCTYPE html> - w HTML (w wersji 5) jest to oznajmienie przeglądarce, że dokument, który będzie interpretowany jest właśnie kodem HTML.

<html> … </html> - znaczniki informujące przeglądarkę o tym, że wewnątrz nich znajdują się wszystkie inne znaczniki HTML - zawartością tego taga mogą być jedynie <body> oraz <head>, o których powiemy sobie niebawem.

<head> … </head> - inaczej mówiąc, nagłówek dokumentu HTML - tak jak w wypracowaniu czy eseju, spełnia on bardzo ważną rolę rozpoczęcia. Zawiera w sobie inne znaczniki takie jak tytuł dokumentu, czy źródła z jakich korzysta. W dalszej części kursu opowiemy w jaki sposób dodawać do niego zewnętrzne pliki, czcionki czy też znaczniki typu meta. Ten tag nie służy do umieszczania w nim treści strony.

<body> … </body> - "ciało" dokumentu - to tutaj będziemy umieszczać całą naszej treść strony internetowej.

<h1> … </h1> - jest to jeden z kolekcji znaczników tekstu, które określają nagłówki w dokumencie HTML.

<p> … </p> - znaczniki, które spełniają rolę paragrafu, czyli inaczej adnotacji do tekstu.

O znacznikach HTML, możesz poczytać na niżej wymienionych stronach. Nie będziemy rozwijać tego tematu tak bardzo szeroko jak Javy, ponieważ jest to tylko dodatek pozwalający na pełną wizualizację tworzenia aplikacji webowych. Jeżeli poczułbyś, że to co znajduje się w tym module jest dla Ciebie interesujące, możesz zerknąć na ten kurs front-endowy: https://developer.mozilla.org/en-US/docs/Learn/Getting_started_with_the_web/HTML_basics

CSS, czyli zestaw kredek i flamastrów

Cascading Style Sheets (pol. kaskadowe arkusze stylów), czyli język służący do upiększania stron WWW - to dzięki niemu strony nabierają kolorów, są ładnie rozmieszczone czy też są responsywne (dopasowują się do wielkości okna przeglądarki). Można potraktować ten język jak kredki i flamastry dla programisty.

Spójrzmy na przykładowy kod CSS:

body {
   background-color: grey;
}

h1 {
   text-align: center;
}

Przykładowy kod CSS odnosi się bezpośrednio do znaczników HTML takich jak <body> i <h1>. Body oraz h1 są tak zwanymi w języku CSS - selektorami to bardzo ważna część tego języka, ponieważ pozwala na pokazanie, który element ma otrzymać style, a profesjonalnie mówiąc właściwości zawarte pomiędzy nawiasami klamrowymi - { … }. Wewnątrz tych nawiasów możemy zauważyć właściwości takie jak:

  1. background-color - jak sama nazwa wskazuje jest to określenie koloru tła naszej strony
  2. text-align - jest to ułożenie wszystkich tagów <h1> w samym środku strony.

Każda z właściwości posiada wartość, która jest przypisywana po wcześniejszym wstawieniu znaku - : - następnie wpisujemy wartość.

Zazwyczaj edytory podpowiadają możliwe wartości do wpisania wewnątrz właściwości, jednak jeżeli jesteś zainteresowany/a wszystkimi możliwościami warto odwiedzić stronę taką jak: https://developer.mozilla.org/en-US/docs/Learn/CSS/Introduction_to_CSS

Podsumowując - selektor to element, który chcemy ostylować, a za nim w nawiasach klamrowych umieszczamy style, zgodnie z wzorem właściwość: wartość; pamiętając o średniku na końcu każdej linii z właściwością CSS.

Element head

Mamy już wszystko co jest nam potrzebne, by móc omówić co jest zawarte w powyższym kodzie - rozpoczniemy tłumaczenie od zawartości znacznika :

  1. <meta…> - jest to część meta znaczników (opisujących samą stronę), które można traktować jako opis dokumentu HTML. Ich zadaniem jest poinformowanie przeglądarki o rodzaju formatowania tekstu, dostosowaniu się do różnych ekranów, określenie w jakim języku jest prezentowana treść strony itp. Więcej możesz dowiedzieć się pod tym adresem: https://developer.mozilla.org/pl/docs/Web/HTML/Element/meta
  2. <title> - Tytuł strony wyświetlany w pasku kart w przeglądarce
  3. <link…> - zadaniem tych znaczników jest dołączanie innych plików do dokumentu HTML, np. plików CSS bądź czcionek (drugi znacznik )

Element body

Sekcja <body> zawiera wszystko co niezbędne dla naszego widoku, czyli całą treść strony. Zapewne widzisz tu wiele nowych znaczników oraz ich atrybutów (takich jak class, name etc.). Skupimy się na samym początku na tym, co znajduje się wewnątrz znaczników, aby lepiej zrozumieć to, co znajduje się w poniższym kodzie:

Każdy ze znaczników HTML może posiadać wewnątrz siebie atrybuty takie jak np. class, którego służy jako odniesienie dla pliku CSS. Dzięki nadawaniu znacznikom konkretnych klas (nie myl z klasami w Javie) możemy napisać jeden zestaw stylów (CSS) dla wielu znaczników HTML, co znacznie zmniejsza ilość kodu w pliku zawierającym kod CSS - jest to bardzo proste - wystarczy zamiast nazwy elementu (np. "body {}") w selektorze, umieścić tam nazwę klasy, którą mu przypisaliśmy, poprzedzając ją kropką - np. ".someclass {}".

Wewnątrz HTML możemy znaleźć znaczniki, które posiadają swoje własne szczególne atrybuty, które można zastosować - a tym znacznikiem jest na przykład - <form>, który jest reprezentacją formularza na stronie internetowej. Zawiera on 2 kluczowe atrybuty:

  1. method - Jest to metoda HTTP wysłania żądania do serwera aplikacji. W naszym wypadku jest to metoda POST, ponieważ formularz odpowiada za utworzenie nowego zadania. Konsekwencją wysyłanie żądania przy pomocy znacznika <form>jest przeładowanie strony. W dalszej części modułu poznasz technikę wysyłania żądań dynamicznie, bez przeładowania.
  2. action - jest to adres, pod który ma zostać wysłane żądanie HTTP. W tym wypadku jest to adres naszego REST API, które odpowiada za tworzenie nowego zadania w bazie danych.

fieldset

Znacznik ten można potraktować jako znacznik grupujący, ponieważ określa konkretną grupę znaczników pól. Zauważ, że zawiera w sobie znacznik <label> i znacznik <input>. Znacznik <label> pełni funkcję opisującą pola do wprowadzania danych. Poniżej dowiesz się dokładnie czym jest znacznik <input> i jak można go wykorzystać

input

Kolejnym przypadkiem znacznika, które posiada specyficzne atrybuty i może zostać znaleziony w naszym pliku HTML jest <input> - znacznik pozwalający na wprowadzanie danych przy pomocy formularza. Dzięki niemu, użytkownik może wpisać dowolną wartość w polu przeglądarki, a następnie wysłać ją na serwer. Przykładowy input w naszej aplikacji wygląda następująco:

Wewnątrz znacznika <input> mogą znajdować się następujące atrybuty:

  1. type - jest to atrybut specyficzny dla znacznika <input> jego zadaniem jest określenie jakiego rodzaju znacznika <input> potrzebujemy. Najbardziej popularnymi typami są: button (przycisk), text (pole tekstowe do wypełnienia), email (pole do wpisania adresu email z jego walidacją), checkbox, file (pole do dodania pliku), czy submit (zatwierdzenie formularza i wysłanie żądania)
  2. name - jest to atrybut specyficzny dla znaczników, które znajdują się wewnątrz znacznika <form>. Można potraktować go jako zmienną, do której w momencie wysłania żądania do serwera jest przypisywana wartość wprowadzona w formularzu.
  3. placeholder - z tego atrybutu możemy skorzystać kiedy na przykład nasz znacznik <input> korzysta z atrybutu type=”text”, wtedy wewnątrz pola do wpisywania tekstu znajdzie się tekst tymczasowy definiujący chociażby jakich danych oczekuje od użytkownika:

Atrybuty sprawdzające - required i min-length

Wewnątrz znaczników typu <input> możesz znaleźć również atrybuty takie jak required (oznaczenie pola jako wymagane), czy min-length (oznaczenie minimalnej długości tekstu podawanego w polu minimalna długość) - są to znaczniki sprawdzające, pozwalające na wstępną walidację wprowadzonych danych. Więcej atrybutów sprawdzających możesz poznać pod adresem: https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/HTML5/Constraint_validation

section

Znacznik opisujący sekcję dokumentu. W dobrym wypracowaniu można podzielić dokument na sekcje, na przykład tematyczne. Jest to zabieg, który pozwala na lepsza organizację dokumentu. Wpływa na jego czytelność, oraz pozwala na nałożenie styli na określoną grupę znaczników zawartych w znaczniku <section>.

div

Jest to znacznik, który podobnie jak <section> wydziela dokument HTML na części. Często stosuje się go do dużo mniejszych grup niż <section>. Można go zastosować do grupowania 2 elementów, które na stronie powinny być ułożone w kompletnie inny sposób niż pozostałe elementy.

Ostatnim ciekawym dla nas znacznikiem może być znacznik <script></script>, który znajduje się na samym dole dokumentu HTML i zawiera w sobie pliki o rozszerzeniu .js. Znacznik ten odpowiada za dołączanie do dokumentu HTML plików z kodem JavaScript. Ale o tym już wkrótce!

20.3. JavaScript i jQuery

Moduł ten poświęcimy zrozumieniu i poznaniu podstaw języka JavaScript oraz bardzo popularnej bibliotece jQuery ułatwiającej pisanie w języku JavaScript. Rynek oferuje masę różnych bibliotek wspomagających pisanie w języku JavaScript, a każda nich posiada własne podejścia do różnych problemów.

Najbardziej popularnymi bibliotekami są Angular oraz ReactJS, jednakże są to już tak duże narzędzia, że próg wejścia i zrozumienia ich jest równorzędny ze zrozumieniem Javy i Springa. Z racji tego, że zależy nam przede wszystkim na wizualizacji i zrozumieniu komunikacji warstwy frontendowej z warstwą backendowa, zdecydowaliśmy się na użycie biblioteki jQuery, której zrozumienie jest stosunkowo proste i wciąż powszechnie wykorzystywane.

Czym jest JavaScript?

JavaScript jest językiem interpretowanym działającym po stronie przeglądarki, czyli językiem, który nie wymaga kompilacji jak to występuje w przypadku Javy za którą zapewne bardzo tęsknisz :)

Interpretowany język to taki który w formie w której piszesz jest już rozumiany przez odpowiednio stworzone programy - możesz porównać sobie to z uruchamianiem kodu od razu na Java Virtual Machine, bez wcześniejszej kompilacji.

JavaScript to język sprawdzający się idealnie w sytuacjach, kiedy na stronie oczekujemy dynamicznych reakcji. Przypomnij sobie przypadki w których nagle na stronie internetowej pojawiają się nowe elementy dosłownie w chwile po kliknięciu przycisku - odbywają się to właśnie dzięki temu językowi.

Dobrym przykładem zastosowania języka JavaScript jest przeglądanie tablicy na Facebooku. Wyobraź sobie, że znajdujesz interesujące Cię zdjęcie, wybierasz je przyciskiem myszy i nagle tło strony zaczyna być mniej wyraźne, zdjęcie się powiększa, a po prawej stronie widzisz dokładnie zapisane komentarze. Jak myślisz, czyja to zasługa? :)

JavaScript pozwala również na wysyłanie żądań do serwerów HTTP w sposób dynamiczny, bez przeładowania strony. Dzięki temu rozwiązaniu jesteśmy w stanie tworzyć dynamiczne widoki nie odrywając uwagi użytkownika od pracy / zabawy czy też zwykłego przeglądania wiadomości i to ta funkcjonalność będzie interesować nas najbardziej, ponieważ zobrazuje nam komunikację dwóch warstw - frontendowej i backendowej.

Tworzenie podstawowego skryptu

Poznajmy trochę podstaw kodu JavaScript. Kod może być znajdować się bezpośrednio w pliku o rozszerzeniu .js, bądź znajdować się bezpośrednio wewnątrz pliku .html. W celach edukacyjnych wykorzystamy połączony kod HTML wraz z kodem JavaScript, jednakże później będziemy stosować tylko i wyłącznie kod wyodrębniony do osobnych plików .js.

Utwórz nowy plik HTML o nazwie simple_page.html w edytorze Sublime Text:

<!DOCTYPE html>
<html>
<body>

<h1>Kodilla First Java Script Code</h1>

<button id="button-1">I am waiting</button>

<script>
 var button = document.getElementById('button-1');

 button.addEventListener('click', function() {
   button.innerHTML = 'I am clicked';
 });

</script>
</body>
</html>

W powyższym przykładzie pojawiło się kilka nowych rzeczy. Najważniejsze z nich to:

  • document - obiekt reprezentujący stronę internetową w JavaScripcie - jest on obecny domyślnie na każdej stronie internetowej. Udostępnia nam on m.in. metody do sprawdzania pod jakim adresem internetowym (URL) się znajdujemy czy do wyszukiwania elementów HTML na stronie.
  • document.getElementById(id) - o wilku mowa - wyszukiwanie elementów na stronie. Ta metoda przyjmuje jako argument string reprezentujący id (identyfikator) elementu HTML i zwraca nam ten element w formie obiektu JavaScriptowego. Warto wspomnieć, że identyfikatory elementów muszą być unikalne i niepotwarzalne na stronie, więc ta metoda zawsze zwróci nam jeden element. W naszym przykładzie przypisaliśmy sobie ten element do zmiennej, aby później z niego korzystać. Zwróć uwagę na to, że umieściliśmy id, które jest stringiem w apostrofach (single quotes) zamiast w cudzysłowach - w JavaScripcie używanie apostrofów i cudzysłowów do oznaczania stringów jest wymienne - można używać jednej bądź drugiej opcji i nie ma to wpływu na działanie kodu.
  • button.addEventListener(eventName, callback) - elementy HTML które przechowujemy w formie obiektów JavaScript również udostępniają nam różne metody i właściwości - np. metodę addEventListener, która jako argumenty przyjmuje nazwę zdarzenia oraz funkcję, która ma zostać wykonana kiedy to zdarzenie wystąpi na danym elemencie. Zdarzenie 'click' z przykładu oznacza kliknięcie na element - w tym przypadku na przycisk o id "button-1".
  • button.innerHTML - skoro wspomnieliśmy o właściwościach elementów HTML, pora przedstawić jedną z nich - innerHTML. Ta właściwość zwraca nam string zawierający całą treść elementu HTML na którym została wywołana - możemy też ustawić jej wartość, co zmieni zawartość danego elementu na to, co ustawimy. W naszym przypadku ustawiamy jako treść przycisku tekstu 'I am clicked!'.

Całość umieściliśmy w tagu <script></script>, który dodaliśmy do kodu HTML tuż przed zamknięciem - to istotne, aby skrypty JavaScript zawsze dołączać do stron w ten sposób - dzięki temu, przeglądarka najpierw ładuje style oraz kod HTML stanowiący treść strony, a później zajmuje się kodem JavaScript. Dlaczego nie warto pobrać kodu JavaScript najpierw? Otóż kod JavaScript zazwyczaj działa w oparciu o elementy HTML i zdarzenia które na nich zachodzą (np. przewijanie strony, kliknięcie w element) - dopóki te elementy się załadują, to kod JavaScript i tak nie miałby co robić :)

Aby dostarczyć sobie odrobinę radości, zapisz powyższy kod w swoim pliku HTML, a następnie uruchom przeglądarkę i kliknij przycisk!

Utworzenie prostego dokumentu HTML, korzystającego z jQuery

Utwórzmy teraz folder o nazwie jquery_sample, w którym umieścimy plik index.html o następującym kodzie:

<html>
<head>
  <title>jQuery is awesome</title>
</head>
<body>
  <div id="my_custom_div">
    <p>Hello Everyone!</p>
  </div>

  <button id="my_custom_button"> Toggle it, babe !</button>

  <script src="jquery-3.2.1.min.js"></script>
  <script src="my_jquery_script.js"></script>
</body>
</html>

Następnie, utwórz pusty plik my_jquery_script.js wewnątrz naszego katalogu jquery_sample. Folder powinien wyglądać następująco:

Do swojego pliku my_jquery_script.js dodaj poniższy kod:

$( document ).ready(function() {
    $("#my_custom_button").click(function() {
       $("#my_custom_div").toggle();
    });
});

Omówmy go, bo to kompletna nowość!

Zacznijmy od fragmentu $( document ).ready(function() {}) i od opowiedzenia sobie czym jest ten magiczny $ - jest to skrótowa nazwa obiektu jQuery, który udostępnia nam ta biblioteka. Za jego pośrednictwem możemy np. wybierać elementy podobnie jak robiliśmy to za pomocą document.getElementById, tym razem jednak musimy na początku stringa z id umieścić znak #, który oznacza identyfikator w selektorach CSS oraz przy wybieraniu elementów za pomocą jQuery.

Wyżej wymienione $( document ).ready(function() {}) oznacza funkcję, która zostanie wywołana w momencie, kiedy przeglądarka pobierze całą treść strony - jest to dokładnie to samo, co w przypadku reagowania na zdarzenia za pomocą element.addEventListener w JavaScripcie. W naszym przykładzie chcemy wywołać daną funkcję dopiero wtedy, kiedy cała treść przeglądarki zostanie załadowana. Robimy to z tego samego powodu dla którego umieszczamy pliki JavaScript na samym końcu elementu <body> - treść strony musi się najpierw załadować, aby kod JavaScript miał na czym operować.

Kolejną interesującą linijką jest $("#my_custom_button").click(function() {})- tutaj z kolei wybieramy element o id "my_custom_button" za pomocą obiektu jQuery ($), a następnie wywołujemy na nim metodę click, podając jako argument funkcję, która ma się wykonać po kliknięciu na ten element - jest to jednoznaczne z document.getElementById('my_custom_button').addEventListener('click', function() {}), jednak jest to znacznie krótszy i prostszy kod, prawda? :)

Ostatnią ciekawą rzeczą w tym snippecie jest $("#my_custom_div").toggle() - ta linijka najpierw wybierze element o id "my_custom_div", a następnie ukryje go jeśli jest widoczny (nadając mu styl display: none; lub pokaże go, jeśli jest obecnie niewidoczny. Jest to całkiem przydatne np. w tworzeniu stron, które pokazują dodatkowy opis np. sprzedawanego przedmiotu po kliknięciu na jakiś przycisk.

Jak widzisz, tworzenie prostych interakcji z interfejsem strony staje się znacznie prostsze dzięki jQuery :)

20.4. Wysyłanie danych za pomocą AJAX i wyświetlanie ich na widoku

Witaj w ostatniej części tego modułu! Pora na poznanie sposobu w jaki JavaScript wysyła żądania HTTP do serwera. Poniżej przygotowaliśmy gotowy kod, którego części omówimy i przeanalizujemy tak, abyś rozumiał w jaki sposób front-endowcy będą pisać kod komunikujący się z Twoimi API, które będziesz tworzył w Javie jako programista back-endowy. Poznamy na drodze do osiągnięcia w 100% działającej aplikacji, poznasz także nowe możliwości biblioteki jQuery! Do dzieła! Przekopiuj poniższy kod do pliku script.js i przenieś go na swój serwer, do folderu htdocs:

$(document).ready(function() {

  var apiRoot = 'http://localhost:8080/v1/task/';
  var datatableRowTemplate = $('[data-datatable-row-template]').children()[0];
  var tasksContainer = $('[data-tasks-container]');

  // init
  getAllTasks();

  function createElement(data) {
    var element = $(datatableRowTemplate).clone();

    element.attr('data-task-id', data.id);
    element.find('[data-task-name-section] [data-task-name-paragraph]').text(data.title);
    element.find('[data-task-name-section] [data-task-name-input]').val(data.title);

    element.find('[data-task-content-section] [data-task-content-paragraph]').text(data.content);
    element.find('[data-task-content-section] [data-task-content-input]').val(data.content);

    return element;
  }

  function handleDatatableRender(data) {
    tasksContainer.empty();
    data.forEach(function(task) {
      createElement(task).appendTo(tasksContainer);
    });
  }

  function getAllTasks() {
    var requestUrl = apiRoot + 'getTasks';

    $.ajax({
      url: requestUrl,
      method: 'GET',
      success: handleDatatableRender
    });
  }

  function handleTaskUpdateRequest() {
    var parentEl = $(this).parent().parent();
    var taskId = parentEl.attr('data-task-id');
    var taskTitle = parentEl.find('[data-task-name-input]').val();
    var taskContent = parentEl.find('[data-task-content-input]').val();
    var requestUrl = apiRoot + 'updateTask';

    $.ajax({
      url: requestUrl,
      method: "PUT",
      processData: false,
      contentType: "application/json; charset=utf-8",
      dataType: 'json',
      data: JSON.stringify({
        id: taskId,
        title: taskTitle,
        content: taskContent
      }),
      success: function(data) {
        parentEl.attr('data-task-id', data.id).toggleClass('datatable__row--editing');
        parentEl.find('[data-task-name-paragraph]').text(taskTitle);
        parentEl.find('[data-task-content-paragraph]').text(taskContent);
      }
    });
  }

  function handleTaskDeleteRequest() {
    var parentEl = $(this).parent().parent();
    var taskId = parentEl.attr('data-task-id');
    var requestUrl = apiRoot + 'deleteTask';

    $.ajax({
      url: requestUrl + '/?' + $.param({
        taskId: taskId
      }),
      method: 'DELETE',
      success: function() {
        parentEl.slideUp(400, function() { parentEl.remove(); });
      }
    })
  }

  function handleTaskSubmitRequest(event) {
    event.preventDefault();

    var taskTitle = $(this).find('[name="title"]').val();
    var taskContent = $(this).find('[name="content"]').val();

    var requestUrl = apiRoot + 'createTask';

    $.ajax({
      url: requestUrl,
      method: 'POST',
      processData: false,
      contentType: "application/json; charset=utf-8",
      dataType: 'json',
      data: JSON.stringify({
        title: taskTitle,
        content: taskContent
      }),
      complete: function(data) {
        if(data.status === 200) {
          getAllTasks();
        }
      }
    });
  }

  function toggleEditingState() {
    var parentEl = $(this).parent().parent();
    parentEl.toggleClass('datatable__row--editing');

    var taskTitle = parentEl.find('[data-task-name-paragraph]').text();
    var taskContent = parentEl.find('[data-task-content-paragraph]').text();

    parentEl.find('[data-task-name-input]').val(taskTitle);
    parentEl.find('[data-task-content-input]').val(taskContent);
  }

  $('[data-task-add-form]').on('submit', handleTaskSubmitRequest);

  tasksContainer.on('click','[data-task-edit-button]', toggleEditingState);
  tasksContainer.on('click','[data-task-edit-abort-button]', toggleEditingState);
  tasksContainer.on('click','[data-task-submit-update-button]', handleTaskUpdateRequest);
  tasksContainer.on('click','[data-task-delete-button]', handleTaskDeleteRequest);
});

Omówienie kodu zawartego w script.js

Zmienne

W języku JavaScript, tak samo jak w Javie, możemy zapisywać zmienne, które wykorzystamy później w kodzie. Poniżej widzimy 3 zmienne, których przeznaczenie opiszemy poniżej:

  1. var apiRoot - jest to zmienna, która przechowuje adres naszej aplikacji backendowej. Niezbędny do komunikacji warstwy frontendowej z warstwą backendową.
  2. var datatableRowTemplate - zmienna, do której pobierany jest szablon wiersza tabeli wyświetlającej dodane zadania. Przyda nam się wtedy, gdy będziemy dodawać do niej pobrane zadania. Możesz skojarzyć to sobie z wzorcem projektowym Prototype, w którym tworzyliśmy skomplikowane i powtarzalne części aplikacji tylko raz, a następnie uzupełniliśmy je danymi.
  3. var tasksContainer - zapisanie referencji do elementu HTML, w którym będziemy 'rysować' wszystkie zadania za pomocą JavaScriptu.

Funkcje

W języku JavaScript możemy również używać funkcji! Pomagają w organizacji kodu. Poniżej prezentujemy inicjalizacyjne wywołanie funkcji, która wysyła żądanie o pobranie wcześniej dodanych zadań do naszej bazy danych.

W JavaScripcie funkcje tworzymy za pomocą poniższego schematu:

Funkcja na poniższym screenshocie wykonuje żądanie do naszej aplikacji w Javie, której celem jest pobranie istniejących już zadań w bazie danych:

Funkcja getAllTask odpowiada za wysłanie żądania do naszej backendowej aplikacji, która zwróci zapisane wcześniej w bazie zadania. Wewnątrz metody możemy zauważyć przypisanie wartości do zmiennej. W trakcie przypisania występuje konkatenacja znana już Tobie z Javy. Do podstawowego adresu URL przypisujemy wyspecyfikowaną ścieżkę do pobrania zadań.

Wysłanie żądania typu GET

Wysyłanie żądania w bibliotece jQuery występuje poprzez wywołanie metody ajax, która jako argument przyjmuje JSON o różnych wartościach niezbędnych do wysyłania żądania. W trakcie tego submodułu poznasz różne konfiguracje i dostosowania do żądania GET i POST.

Pierwszym i najważniejszym parametrem dla metody ajax udostępnianej przez jQuery jest adres URL, pod który została podstawiona zmienna apiRoot utworzona wcześniej. W dalszej części jesteśmy zobowiązani do podania rodzaju żądania - w naszym wypadku do pobrania zadań jest to GET. Możemy powiedzieć metodzie $.ajax co dokładnie ma się stać, kiedy żądanie zakończy się otrzymaniem odpowiedzi. My podajemy funkcję handleDatatableRender, która znajduje się poniżej:

Funkcja przyjmuje argument data, który w rzeczywistości jest odpowiedzią z serwera (o cały proces odbierania żądania dba jQuery).

W pierwszej kolejności odbywa się wywołanie funkcji oczyszczenia kontenera naszych zadań, który może zawierać wcześniej stworzone zadania. Następnie odbywa się przejście w pętli po tablicy otrzymanych zadań, oraz wywołanie funkcji utworzenia elementu HTML z danymi otrzymanymi z serwera. Po jego utworzeniu na danym elemencie HTML metoda appendTo przekazuje element do kontenera zadań, który pokaże zadania na stronie internetowej.

Wysyłanie żądania typu POST

Przed nami nowy rodzaj żądania - POST - funkcja handleTaskSubmitRequest zajmuje się wysyłaniem nowo utworzonego zadania w warstwie front-endowej do warstwy backendowej. Przyjmuje argument event, który jest przesyłany do funkcji kiedy użytkownik użyje przycisku create w widoku aplikacji.

Funkcja rozpoczyna się od użycia metody event.preventDefault(), która w tym przypadku zapobiega przeładowaniu strony po wysłaniu formularza z danymi nowego zadania. Obiekt event jest przekazywany do naszej funkcji, ponieważ wcześniej dowiązaliśmy do formularza dodawania postów funkcję reagującą na zdarzenie wysłania formularza - odpowiada za to następujący kod - $('[data-task-add-form]').on('submit', handleTaskSubmitRequest);

Funkcje dowiązane do zdarzenia zawsze otrzymują parametr event, który zawiera dane opisujące zdarzenie, które zaszło - element który je wywołał, typ tego zdarzenia oraz wiele innych rzeczy. Zawiera on też wspomnianą wyżej metodę preventDefault(), która zapobiega domyślnej reakcji przeglądarki na dane zdarzenie (np. przewinięcia strony w dół lub w górę na zdarzenie scroll), co pozwala nam zastąpić je naszymi własnymi reakcjami - w tym przypadku jest to wysłanie formularza do naszego API bez przeładowania strony.

Następnie, zbieramy informacje z pól znajdujących się na widoku w części przeznaczonej do utworzenia nowego zadania - informacje te zostają przypisane do zmiennych: taskTitle i taskContent. Po inicjalizacji zmiennych, następuje utworzenie adresu, pod który zostanie wysłane żądanie mające na celu utworzenie zadania w bazie danych.

Po wykonaniu wszystkich kluczowych czynności, które przygotowują nam dane do wysłania do API, przychodzi kolej na realizację żądania przy pomocy poznanego już przez Ciebie ajax:

  1. url - adres URL pod który należy wysłać żądanie
  2. method - metoda HTTP jaką ma zostać wysłane żądanie
  3. processData - tutaj określamy czy jQuery ma formatować dane, które przekazujemy w data w jakikolwiek sposób - zaznaczamy to jako false, ponieważ robimy to sami.
  4. contentType - rodzaj żądania jaki przesyłamy do serwera. Za jego pomocą informujemy serwer, że dane wysłane przez nas są w formacie JSON.
  5. data: JSON.stringify - w tym miejscu następuje transformacja danych żądania do typu String.
  6. complete - dzięki temu polu możemy ustalić co stanie się po poprawnym wykonaniu żądania. W naszym wypadku sprawdzamy czy serwer zwrócił odpowiedni status odpowiedzi, a następnie pobieramy na nowo zadania, aby wyświetlić nowe zadanie na widoku.
